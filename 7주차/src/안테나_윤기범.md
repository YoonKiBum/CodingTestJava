### 유형 : 그리디
### 윤기범
https://www.acmicpc.net/problem/18310

### 문제 접근 방식
  - 문제를 읽고 그리디로 판단함
  - 1 <= n <= 20,0000 2중 for문 불가능
  - 문제를 예시를 잘 살펴보면 오름차순으로 정렬한 후  가운데값에 설치하는것이 이득임을 알 수 있음
  - 단, n이 짝수인 경우는 n / 2 - 1, 홀수인 경우는 n / 2이다.
  ```java
  Arrays.sort(arr);
  int point = 0;
  if(n % 2 == 0) {
    point = n / 2 - 1;
  } else {
    point = n / 2;
  }
  ```
